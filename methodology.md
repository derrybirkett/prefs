# Development Methodology

## Learning Organizations

I value environments that promote continuous learning, knowledge sharing, and collective improvement. Systems should be designed to capture learnings and make them accessible to all team members (human and AI).

## Lean Methodology

Development follows lean principles:

### Small Iterative Improvements
- Break work into small, testable increments
- Each iteration should deliver measurable value
- Fail fast, learn quickly, adapt continuously

### Visionary Model
- Maintain a clear long-term vision
- Let the vision guide prioritization
- Don't sacrifice the vision for short-term convenience
- Iterate towards the vision, not away from it

### Waste Reduction
- Eliminate work that doesn't serve the user or vision
- Automate repetitive tasks
- Document once, reuse everywhere

## Design Thinking

Apply design thinking principles throughout development:

1. **Empathize**: Understand user needs and context
2. **Define**: Clearly articulate the problem to solve
3. **Ideate**: Explore multiple solution approaches
4. **Prototype**: Build minimal versions to test ideas
5. **Test**: Validate with real users and iterate

Design thinking applies to both product features and development tooling.

## Product Development Agent Stack

Development should be orchestrated by a logical stack of specialized agents:

- **Strategy Agent**: Aligns work with vision and goals
- **Design Agent**: Focuses on user experience and interface
- **Implementation Agent**: Writes production code
- **Testing Agent**: Ensures quality and validates flows
- **Documentation Agent**: Maintains docs, changelogs, and blogs
- **DevOps Agent**: Handles deployment and infrastructure

Each agent has clear responsibilities and interfaces with others through well-defined contracts (issues, PRs, test results, documentation).
